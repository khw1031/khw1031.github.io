## 아이디어

셋로그(Setlog)는 굿즈 서비스가 아니라 **여러 명이 함께 하루를 기록하는 협업 브이로그 앱**이다 — 3인이 시간 알람에 맞춰 각자 짧은 영상을 찍으면 자동으로 한 편의 분할 화면 브이로그로 합쳐지고, 완성본을 릴스·틱톡·쇼츠로 공유한다. 이 아이디어의 진짜 레퍼런스는 셋로그의 **제품**이 아니라 그 **메커닉** — "여러 사람이 하나의 결과물을 공동 생성 → 자동/반자동 완성 → SNS 네이티브 공유" — 이고, 이걸 **굿즈 온디맨드 커머스**에 이식한 것이다.

구체적으로는 특정 굿즈(티셔츠·컵·폰케이스 등)를 두고 **여러 인원이 방에 들어와 동시 또는 비동기로 함께 편집**한다(친구끼리 커플/모임/팬덤 굿즈를 같이 꾸미는 그림). 편집이 끝나면 그 디자인을 **POD(주문형 인쇄) 업체에 프린팅 주문 → 배송**으로 흘려보내 판매한다. 사용자가 제안한 역할 분담을 그대로 옮기면: ==결제·주문·풀필먼트 연동은 Backend, 협업 편집·이미지 처리·SNS 공유 트리거는 Lead, 미리보기·이미지 다운로드는 Frontend==. 마플 같은 기존 POD가 **1인 커스텀 에디터**라면, 이 아이디어의 차별점은 **협업(멀티플레이어) 편집 + 소셜 공유 루프**를 커머스의 앞단에 두는 것이다.

## 더 해볼 질문

- **차별점의 핵심이 "협업"인가 "커머스"인가**: 마플·Printful은 이미 POD를 잘한다. 이 아이디어가 이기는 지점은 "여럿이 함께 만든다"는 협업/소셜 경험이어야 하는데, 그게 실제 결제로 이어지는 훅인지, 아니면 재미 요소에 그치는지부터 검증해야 한다.
- **동시편집 vs 비동기편집**: 실시간 멀티플레이어 캔버스(Figma형 라이브 커서·충돌 해결)는 기술 난도·비용이 크다. 초기엔 비동기(순번/레이어 잠금/코멘트) 협업으로 충분한지, 실시간이 정말 핵심 가치인지 결정.
- **누가 사나 / 누가 값을 내나**: 함께 만든 굿즈를 (a) 참여자 각자가 자기 것 1개씩 사는가, (b) 대표 1인이 여러 개 주문하는가, (c) 제3자에게 판매하는가? 결제 주체·수량 모델이 UX와 단가를 좌우한다.
- **POD를 직접 하나, 연동하나**: Printful/Printify API로 프린팅·배송을 위탁(가볍게 시작, 마진 얇음)할지, 마플처럼 자체 생산(무겁지만 마진·품질 통제)할지. 초기엔 API 위탁이 합리적으로 보이나 확인 필요.
- **SNS 공유가 성장 루프가 되나**: 셋로그의 힘은 "완성본을 자동으로 공유 가능한 콘텐츠로" 만든 데 있다. 협업 굿즈도 완성 시 공유용 목업/영상을 자동 생성해 바이럴 루프를 만들 수 있는가?
- **IP·저작권·부적절 이미지**: 여럿이 올린 이미지의 저작권/초상권, 상표·부적절 콘텐츠 필터링 책임은 누구에게? POD 위탁 시 업체 정책과 충돌 지점 확인.
- **최소 검증(MVP)**: 실시간 협업·자체 POD 없이 "비동기 협업 편집 + Printful 연동 + 자동 공유 목업" 조합으로 수요를 먼저 검증할 수 있는가?

## 레퍼런스·서비스

_확인일 2026-07-10. 벤더/공식 사이트는 자기 서비스 주장에 한해 1차, 비교·집계·수치는 2차._

- **셋로그(Setlog)** — 이 아이디어가 빌려온 원본 메커닉. "Set(3인)+Log(기록)": 여러 명이 알람에 맞춰 각자 촬영 → 자동 분할 화면 편집 → 릴스/쇼츠 공유. **굿즈가 아니라 협업+자동완성+SNS 공유 패턴**의 레퍼런스. 서비스 1차: [App Store setlog](https://apps.apple.com/kr/app/setlog/id6587576438), 설명 2차: [나무위키 셋로그](https://namu.wiki/w/%EC%85%8B%EB%A1%9C%EA%B7%B8)
- **마플(Marpple)** — 국내 대표 POD 플랫폼. 상품 선택 → 이미지 업로드 → 3분 커스텀, 1개부터 주문, 실시간 미리보기, 티셔츠·컵·폰케이스·텀블러 등. **"1인 커스텀 에디터 + 자체 생산 POD"의 기준선** — 이 아이디어의 협업/소셜 차별점을 가늠할 비교군. 서비스 1차: [marpple.com](https://www.marpple.com/en/)
- **MarppleShop (Shopify 앱)** — 마플의 판매자용: 주문 확인·제작·배송·재고까지 대행하는 POD 판매 연동. "커머스 뒷단을 통째로 위탁"하는 형태의 사례. 서비스 1차: [Shopify App Store MarppleShop](https://apps.shopify.com/marppleshop)
- **Printful API** — REST/JSON POD API. 주문 생성·풀필먼트 확정, 목업 생성, 배송·주문 상태 webhook. **"POD를 직접 안 하고 API로 위탁"하는 MVP 경로**의 후보. 서비스 1차: [printful.com/api](https://www.printful.com/api), 문서 1차: [developers.printful.com](https://developers.printful.com/docs/)
- **Printify API** — 카탈로그 상품에 이미지 얹어 판매·풀필먼트, OAuth 2.0로 머천트 앱 제공. Printful의 대안 POD 백엔드. 문서 1차: [developers.printify.com](https://developers.printify.com/)
- **Figma (멀티플레이어 캔버스)** — 브라우저 실시간 공동 편집·라이브 커서·동기화 선택의 사실상 표준. "동시편집"을 진짜로 하려 할 때의 기술 기준선(난도·비용의 상한선 감). 서비스 1차: [figma.com](https://www.figma.com/), 비교 2차: [Figma vs Canva vs Adobe 비교](https://saascompared.io/blog/best-design-tools/)
