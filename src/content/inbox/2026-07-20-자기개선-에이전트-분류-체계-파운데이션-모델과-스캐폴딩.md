## TL;DR
- 에이전트의 자기개선은 **모델 가중치를 바꾸는 축**과 **주변 스캐폴딩을 바꾸는 축**으로 분리되며, 두 축을 조합하면 현대 agentic 시스템 대부분의 학습 루프를 한 장으로 그릴 수 있다.

## 큰 그림
```
Self-Improving Agent
│
├── 🧠 Foundation Model Improvement (파라미터 θ 업데이트)
│   ├── 1.1 Intrinsic Generative Demonstrations  ── 자기생성 데이터로 SFT
│   ├── 1.2 Intrinsic Evaluative Feedback        ── 자기평가 보상으로 RL
│   └── 1.3 Extrinsic Exploratory Experience
│       ├── 1.3.1 Grounded Task Env. (웹·로봇·코드)
│       └── 1.3.2 Simulated Proxy Env. (World Model)
│
└── 🏗️ Scaffolding Improvement (θ 고정, 외부 컴포넌트 갱신)
    ├── 2.1 Prompt Optimization
    │     ├─ Scalar Feedback
    │     ├─ Qualitative Feedback
    │     ├─ Population-Based Evolution
    │     └─ Textual Gradient
    ├── 2.2 Memory (Object / Structure / Processing)
    ├── 2.3 Tool (Routing / Iterative Refinement / Autonomous Creation)
    └── 2.4 Full Scaffolding (Gödel Machine 계열, 그래프·코드 자체 재작성)
```

## 핵심
- 원문은 **FM-based agent = Foundation Model + Scaffolding**이라는 형식적 추상화에서 출발한다. 즉 LLM 자체와 그것을 감싼 prompt·memory·tool·제어 로직을 명시적으로 분리한 뒤, **"무엇이 업데이트되는가"**로 자기개선을 두 축으로 가른다. 이 분리는 단순 분류를 넘어, 파라미터 학습이 불가능한 폐쇄형 API(Claude, GPT 등)에서도 scaffolding 축으로 자기개선이 가능하다는 설계 함의를 낳는다.
- Foundation Model 축은 다시 **신호의 출처**로 나뉜다. 모델이 스스로 생성한 output을 학습 데이터로 재사용하면 1.1, 모델 내부 평가(자신감·일관성)를 보상 삼으면 1.2, 외부 환경과 상호작용한 궤적을 쓰면 1.3이다. 1.3은 다시 실제 환경(grounded)과 세계모델 기반 시뮬레이션(proxy)으로 갈라지는데, 후자는 안전하고 빠른 탐색을 가능하게 하는 대신 **모델 세계와 현실의 괴리**라는 한계를 안는다.
- Scaffolding 축은 파라미터를 건드리지 않으므로 **배포 중 온라인 개선**이 가능하다. Prompt→Memory→Tool→Full Scaffolding 순으로 변경 범위가 커지며, 가장 극단인 2.4(Gödel Machine 계열)는 에이전트가 자신의 소스 코드나 그래프 자체를 재작성한다.

## 깊이
- **[2.1.4 Textual Gradient — TextGrad]** 자연어 피드백을 "기울기"처럼 취급해 변수(프롬프트·코드)를 수정하는 방식. 비유하면 **LLM이 loss 대신 문장 비판을 역전파**하는 것. 다만 진짜 미분 가능한 기울기가 아니므로 수렴 보장이 없고, 비판 품질이 상한선을 결정한다(저자 주장).
- **[2.3 Autonomous Tool Creation — Voyager]** Minecraft에서 에이전트가 스스로 코드로 도구를 만들고 라이브러리에 쌓아가는 설계. 이후 OS-Copilot·Alita·LLM-ToolMaker로 이어지는 "도구를 소비하지 않고 생산하는 에이전트" 계열의 시발점으로 평가된다(원문 분류). 한계는 생성된 도구의 안전성 검증이 사후 테스트에 의존한다는 점.
- **[2.4 Full Scaffolding — Darwin Gödel Machine / Gödel Agent]** 에이전트가 자기 소스 코드를 변이·선택·재작성하는 개방형 진화 루프. STOP(재귀적 자기개선 코드 생성)에서 아이디어를 이어받아 AlphaEvolve·Hyperagents·MOSS로 확장됐다. 재작성 대상이 코드 그 자체이므로 **정지 문제·루프 폭주**가 이론적 경계다(불확실 — 원문에 명시적 분석은 없음).

## 용어 풀이
- **Foundation Model (FM)** — 대규모 사전학습 모델. 비유: 범용 두뇌. 깨지는 점: 두뇌 비유는 학습·기억을 암시하지만, FM은 배포 후 파라미터가 고정되는 경우가 많아 "학습하는 두뇌" 비유는 scaffolding이 없으면 틀리다.
- **Scaffolding** — FM을 감싸는 prompt/memory/tool/workflow. 비유: 무대 발판. 깨지는 점: 발판은 수동적이지만 실제 scaffolding은 자기 갱신하므로 "능동적 껍데기"가 더 정확.
- **Intrinsic vs Extrinsic** — 신호가 모델 내부에서 오느냐(FM의 출력·평가), 외부 환경에서 오느냐. RL 용법과 일치.
- **World Model** — 환경을 내부에 시뮬레이션하는 모델. 비유: 마음속 연습. 깨지는 점: 실제 물리·UI 드리프트를 놓치면 시뮬레이션 편향이 그대로 정책에 전이된다.

## 핵심 시사점 / 판단
- **(저자 주장)** 자기개선은 단일 기법이 아니라 "무엇을 업데이트하는가" × "어떤 신호로 업데이트하는가"의 2차원 설계 공간이며, 실제 시스템은 여러 칸을 조합한다.
- **(저자 주장)** 2024~2025를 거치며 1.2(Intrinsic Evaluative)와 2.4(Full Scaffolding) 논문이 집중적으로 늘었다 — 외부 보상 없이 reasoning을 개선하려는 흐름과 연결된다.
- **(검증 필요·불확실)** 평가 매트릭스 그림(paper-benchmark incidence)은 제공되나 정량적 우위를 주장하는 메타분석은 원문에 없다. 어떤 축이 비용 대비 가장 효과적인지는 미결.
- **(원문에 없음)** 폐쇄형 FM + scaffolding 개선의 실제 산업 배포 사례 및 안전 가드레일에 대한 실증은 본 리스트에 체계적으로 정리돼 있지 않다.
