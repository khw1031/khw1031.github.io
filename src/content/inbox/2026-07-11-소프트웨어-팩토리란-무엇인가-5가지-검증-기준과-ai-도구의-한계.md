## TL;DR
- 소프트웨어 팩토리는 코딩 도구가 아니라, 비즈니스 요구를 입력받아 검증·추적 가능한 완성품을 책임지고 납품하는 **생산 시스템**이다—AI 에이전트는 여기서 '도구'일 뿐이다.

## 큰 그림
```
소프트웨어 팩토리 (60년 역사)
│
├─ 기원 ── 1969 Hitachi "Software Works" → Toshiba, NEC, Fujitsu
│           2004 MS 저서 → 미 공군 Kessel Run
│
├─ 본질 ── "입력 → 생산 → 품질 보증" 체계 (Ford가 차를 팔듯)
│
├─ 5가지 테스트
│   ├─ T1: 비즈니스 의도에서 시작
│   ├─ T2: 지속적 변경 속 일관성 유지 (drift 방지)
│   ├─ T3: 특정인에 의존하지 않음
│   ├─ T4: 모든 산출물 추적 가능
│   └─ T5: 완성품에 대한 책임 소재 명확
│
├─ 팩토리가 아닌 것
│   ├─ 코딩 에이전트 → 도구 (검증·책임 전가)
│   ├─ 오케스트레이션 대시보드 → 감독 도구
│   └─ 벤치마크 → 도구 성능 측정기
│
└─ 핵심 주장 ── 생산 비용 붕괴 시 가치는 "산출물 보증자"로 이동
```

## 핵심
- 저자는 8090 팀이 1,800만 줄의 COBOL/Assembly 레거시 시스템을 40일 만에 10만 개 이상의 영어 규칙으로 역설계한 사례를 들어, 소프트웨어 팩토리의 실체를 보여준다.
- 팩토리의 핵심은 **생산 시스템 전체의 보증**이다. Ford가 렌치가 아닌 완성차를 팔고 리콜까지 책임지듯, 소프트웨어 팩토리는 요구사항→코드→테스트→배포 전 과정의 무결성을 담보해야 한다.
- AI 코딩 에이전트는 코드 생성 속도를 높이지만, 사양 동기화가 없으면 **drift(요구-문서-코드 간 불일치)를 가속**할 뿐이다. 저자는 "40년간 손으로 쌓은 1,800만 줄 문제를 거버넌스 없는 에이전트 군단이 몇 년 만에 만들 것"이라고 경고한다.
- 따라서 팩토리와 도구를 구분하는 결정적 기준은 **"프로덕션에서 시스템이 고장 나면 누가 전화를 받는가"**다.

## 깊이
- **[T2 — drift 방지]** 저자 주장에 따르면, 코드 생성 자체는 기업 SW의 병목이 아니었다. 진짜 병목은 수십 명이 매주 변경하면서 요구·문서·코드·테스트가 서로 어긋나는 drift다. 팩토리는 이 넷을 **단일 거버닝 객체**로 묶어, 요구사항 변경이 코드를 바꾸고 코드 핫픽스가 요구사항을 업데이트하는 **폐루프**를 구현해야 한다.
- **[T3 — 개인 독립성]** Hitachi의 통계적 품질 관리는 "품질은 라인의 속성이지 작업자의 속성이 아니다"라는 원칙이었다. 팩토리는 지식을 개인이 아닌 시스템에 축적하여, 사람이 빠져나가면 아무것도 남지 않는 '영웅 의존' 구조를 탈피한다.
- **[T5 — 책임]** 저자는 AI 도구 계약서의 IP 조항은 수십 페이지인 반면 책임 조항은 "as-is 제공, 검증은 고객 몫" 한 줄이라고 지적한다. 반면 사례로 든 건강보험사는 청구 규칙을 결정론적 프리필터로 전환해 4년간 $20M+ 절감했는데, 이는 작업 수행자가 결과에 책임질 때만 가능한 수치라는 주장이다.

## 용어 풀이
- **Software Factory** — SW를 제조업 공장처럼 체계적으로 생산하는 시스템 / 비유: 자동차 공장. 깨지는 지점: SW는 물리적 재고가 없고 '완성'의 경계가 모호함.
- **Drift** — 요구사항·문서·코드·테스트 간 불일치가 시간에 따라 누적되는 현상 / 비유: 복사할 때마다 살짝 번지는 복사본. 깨지는 지점: 의도적 변경과 우발적 불일치를 구분하기 어려움.
- **Lot Number(추적성)** — 제조업에서 부품별 배치·라인·교대조를 기록하는 번호 / 비유: 식품 유통이력. SW에서는 요구→코드→테스트→배포 체인이 이에 해당.

## 핵심 시사점 / 판단
- **(저자 주장)** AI 시대 소프트웨어 생산 비용이 붕괴되면, 가치는 "도구 제공자"가 아닌 "산출물 보증자"로 이동한다.
- **(저자 주장)** 에이전트 군단에 거버넌스가 없으면 40년 걸릴 레거시 부채를 수년 만에 축적한다.
- **(검증 필요·불확실)** "1,800만 줄을 40일 만에 10만 규칙으로 역설계"했다는 수치의 재현성과 일반화 가능성은 원문에 근거 없음.
- **(검증 필요·불확실)** 건강보험사 $20M 절감 사례는 단일 사례이며, 팩토리 방식 외 다른 요인이 배제되었는지 불확실.
