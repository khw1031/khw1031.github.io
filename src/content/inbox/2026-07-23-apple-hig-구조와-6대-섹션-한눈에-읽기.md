## TL;DR
- Apple HIG는 "왜(기본 사항) → 어떻게(패턴/구성요소/입력) → 어디서(기술/플랫폼)"로 이어지는 6개 섹션 트리이며, 플랫폼(iOS·macOS·visionOS 등) 공통 원칙을 먼저 세우고 세부 UI 결정을 내리는 구조다.

## 큰 그림
```
                        ┌──────────────────────┐
                        │  Apple HIG (루트)     │
                        └──────────┬───────────┘
                                   │
        ┌──────────┬──────────┬────┴─────┬──────────┬──────────┐
        ▼          ▼          ▼          ▼          ▼          ▼
   [1.시작하기] [2.기본사항] [3.패턴] [4.구성요소] [5.입력] [6.기술]
   (8항목)     (18항목)    (25항목)   (8항목)    (13항목)  (29항목)
        │          │          │          │          │          │
     플랫폼     시각 언어   UX 흐름    UI 부품   조작 방식  시스템 통합
     원칙       (색/글꼴)   (탐색등)   (버튼등)  (터치등)  (Siri등)
```
**흐름**: 설계 철학(1-2)에서 출발 → 사용자 행동 유형(3) → 화면을 채우는 부품(4) → 사용자의 조작 수단(5) → OS/하드웨어 기능 연동(6).

## 핵심
- HIG는 단순 UI 규칙 모음이 아니라 **플랫폼 간 일관성**을 목표로 하는 설계 언어이며, "시작하기"에서 플랫폼별(iOS·macOS·visionOS 등) 디자인 원칙을 먼저 정립한 뒤(원문), 이후 섹션에서 이를 구체적인 요소로 분해한다.
- "기본 사항(Foundations)"은 색상·타이포그래피·레이아웃·머티리얼(Material)·손쉬운 사용(Accessibility)·앱 아이콘 등 **시각·구조의 원자 단위**를 다루며, 이 위에서 "패턴"은 탐색·검색·온보딩 같은 **사용자 여정 단위**로 확장된다(원문).
- "구성요소"는 버튼·메뉴·사이드바 등 **시스템 제공 UI 부품의 사용·커스터마이즈 지침**이고, "입력"은 터치·키보드·제스처 등 **상호작용 채널**, "기술"은 Siri·AR·Apple Intelligence 등 **OS 기능 연동**을 담당한다(원문).
- 최근 업데이트(원문 기준)로 메뉴·스크롤 보기·검색 필드·사이드바·스니펫(Snippets)·Siri 항목이 신규·개정되었는데, 이는 **visionOS·Apple Intelligence 등 신기술 반영** 흐름과 연결되는 것으로 보인다(추론, 원문에 명시적 근거 없음).

## 깊이
- **[기본 사항 — 머티리얼(Material)]**: "Liquid Glass"라는 반투명 레이어 개념이 언급되는데(원문), 이는 배경 콘텐츠를 흐리게 하여 계층(depth)을 표현하는 시각적 소재다. *비유: 트레이싱 페이퍼를 겹쳐 놓은 것.* 깨지는 지점: 실제 물리 유리와 달리 동적 블러·채도 조절이 가능하므로 "유리"라는 단어에 갇히면 안 된다.
- **[패턴 vs 구성요소 구분]**: 패턴은 "무엇을 할 때(작업)" 중심(예: 파일 공유, 온보딩), 구성요소는 "무엇으로(부품)" 중심(예: 툴바, 피커)이다(원문 구조 기반). 이 구분이 무너지면 같은 가이드를 두 곳에서 중복 구현하는 실수가 생긴다.
- **[기술 — Siri]**: 최근 업데이트에 포함된 Siri 항목은(원문) 음성·지능형 제안 등 **OS 수준 통합**을 다루며, 단순 UI를 넘어 앱의 기능이 시스템에 노출되는 방식을 규정한다.

## 용어 풀이
- **HIG (Human Interface Guidelines)** — Apple이 배포하는 UI/UX 설계 지침서 / *비유: 건축의 국가 표준 시방서.* 깨지는 지점: 법적 강제력이 아닌 "가이드라인"이므로 App Store 심사 지침(Review Guidelines)과 혼동하면 안 된다.
- **Foundations(기본 사항)** — 색상·타이포·레이아웃 등 디자인의 원자 요소 / *비유: 레고 블록의 기본 조각.* 깨지는 지점: "기초"라는 말이 "초급자용"으로 오해될 수 있으나, 숙련자에게도 기준점이다.
- **Snippets(스니펫)** — 원문에 항목만 있고 정의가 없어 **원문에 없음(정의)**. 일반적으로는 재사용 가능한 코드/UI 조각을 의미하나 Apple HIG에서의 구체적 의미는 해당 페이지 확인 필요.
- **Liquid Glass** — 머티리얼 섹션에서 언급된 반투명 소재(원문). *비유: 물 위에 뜬 기름막.* 깨지는 지점: 실제 유리가 아니며 OS가 렌더링하는 동적 효과.

## 시각 자료
| 섹션 | 역할 | 항목 수 | 대표 예시 |
|---|---|---:|---|
| 시작하기 | 플랫폼별 설계 원칙 | 8 | iOS·macOS·visionOS 디자인 |
| 기본 사항 | 시각·구조 원자 단위 | 18 | 색상, 타이포그래피, Accessibility |
| 패턴 | 사용자 여정/작업 단위 | 25 | 탐색, 온보딩, 공유 |
| 구성요소 | 시스템 UI 부품 | 8 | 메뉴, 사이드바, 검색 필드 |
| 입력 | 상호작용 채널 | 13 | 터치, 키보드, 포인터 |
| 기술 | OS 기능 연동 | 29 | Siri, Apple Intelligence |

## 핵심 시사점 / 판단
- **(저자 주장)** HIG는 "모든 Apple 플랫폼에서 탁월한 경험"을 위한 지침과 모범 사례를 포함한다(원문).
- **(저자 주장)** 시스템 제공 구성요소를 사용해 "친숙하고 일관적인 경험"을 제공하라고 권장한다(원문).
- **(검증 필요·불확실)** 29개 기술 항목의 구체적 범위(예: Apple Intelligence의 디자인 지침 상세)는 본 landing 페이지에서 확인 불가 — 각 하위 페이지 열람 필요.
- **(불확실)** "Liquid Glass"가 visionOS 전용인지 전 플랫폼 공통인지는 원문 landing page에서 명시되지 않음.
