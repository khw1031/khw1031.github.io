> 한 줄 명제: "랜덤"은 하나가 아니다 — 알고리즘이 계산한 결정론적 유사난수(PRNG)와 물리 현상에서 채취한 진짜 무작위(TRNG/QRNG)는 근본이 다르며, 안전한 보안키는 "충분한 엔트로피(소스) + 역산 불가능성(확장·보관)"을 함께 요구한다. 비결정성이 있다고 좋은 엔트로피 소스는 아니다.

## 큰 그림

```text
Random
├─ 1 PRNG (의사난수, 알고리즘) — 결정론적, seed→동일 수열, 빠름, 재현 가능
│   ├─ 일반 PRNG (LCG, Mersenne Twister, PCG/xorshift)
│   │   └─ 통계적으로만 무작위 → 예측 가능 (역산 쉬움) → 보안용 부적합
│   └─ CSPRNG (ChaCha20, AES-CTR, Fortuna)
│       └─ 결정론적이지만 역산이 "계산적으로 불가능"하게 설계 → 보안용
├─ 2 TRNG (진난수, 물리 엔트로피 채취) — 비결정론적, 재현 불가, 느림
│   ├─ 고전 물리: 열잡음(Johnson–Nyquist), 발진기 지터 → 실질적 무작위
│   └─ QRNG(양자): 방사성 붕괴 시점, 광자 경로 → 근본적(fundamental) 무작위
└─ 3 현실 = 하이브리드
    └─ 물리 엔트로피(TRNG)로 seed → CSPRNG로 확장
        (/dev/urandom, getrandom(), BCryptGenRandom)
```

## 1. PRNG vs TRNG — 무엇이 다른가

- CPU는 결정론적 기계라 순수 소프트웨어로는 진짜 무작위를 못 만든다. 대신 seed에서 함수로 "무작위처럼 보이는" 수열을 계산 → **PRNG**. 같은 seed면 항상 같은 수열(재현 가능 = 장점이자 약점).
- **TRNG**는 계산하지 않고 예측 불가능한 물리 현상(엔트로피 소스)을 측정해서 쓴다. 열잡음은 "변수를 다 몰라서" 예측 못 하는 실질적 무작위, 양자 사건은 물리 법칙 자체가 결과를 확정하지 않는 **근본적** 무작위.

| | PRNG | TRNG |
|---|---|---|
| 근원 | 수학 알고리즘 + seed | 물리 현상 |
| 결정론 | 예 (재현) | 아니오 (재현 불가) |
| 예측 | 원리적으로 가능 | 원리적으로 불가능 |
| 속도 | 매우 빠름 | 느림 |

## 2. PRNG가 보안에 취약한 진짜 이유 — "시도"가 아니라 "계산"

핵심 교정: 취약점은 **브루트포스로 같은 키를 맞히는 것**(예측가능성 ≠ 충돌)이 아니다. 브루트포스는 키 공간이 크면 PRNG든 TRNG든 똑같이 막힌다. PRNG는 **시도 없이 계산으로 뚫린다.**

- **(a) 시드 공간이 좁을 때** — `seed = 현재시각` 같은 실수. 키가 128비트여도 실질 엔트로피는 시드 크기(예: 시간으로 좁혀진 몇백만 개)뿐 → 그 좁은 공간만 뒤지면 재현됨. → 진짜 원인은 "PRNG라서"가 아니라 **엔트로피 부족**.
- **(b) 상태 복원** — 일반 PRNG는 출력 관찰만으로 내부 상태 역산. Mersenne Twister는 **연속 출력 624개**로 상태 완전 복원 → 이후 모든 값 예측. 시도 횟수 0.

→ CSPRNG는 (b)를 막도록 "출력을 봐도 상태를 역산 못 하게" 암호 함수로 설계된 것.

## 3. CSPRNG는 왜 여전히 PRNG인가 — 두 종류의 안전

- **정보이론적 안전성**: 값을 재현할 정보가 원리적으로 없음. 무한 계산력으로도 못 깸. → TRNG/QRNG, One-Time Pad.
- **계산적 안전성**: 미래 값은 내부 상태에 **이미 정해져 있으나**, 출력에서 역산하는 데 우주적 계산량(예: 2^256)이 필요해 현실적으로 불가능. → CSPRNG.

| 구분 | 미래값 정해짐? | 안전 근거 | 예 |
|---|---|---|---|
| 일반 PRNG | 예 | 안전하지 않음(역산 쉬움) | Mersenne Twister |
| CSPRNG | 예 | 계산적 안전 | ChaCha20, AES-CTR |
| TRNG/QRNG | 아니오 | 정보이론적 안전 | 양자 잡음, OTP |

CSPRNG = "물리적으로 진짜 무작위"가 아니라 **"충분히 똑똑한 적도 시간 안에 못 푸는 무작위"**. 실무 보안 대부분이 이 계산적 안전에 의존(OTP는 키 관리 비현실적).

## 4. 안전한 보안키의 조건 — 역산 불가능은 "필요조건일 뿐"

세 가지가 모두 있어야 한다:

1. **충분한 엔트로피** — 키가 담은 실제 불확실성의 양. (a) 시드 부족 사고를 막음. ← TRNG/QRNG/OS 엔트로피 풀이 담당.
2. **역산 불가능성 (backward secrecy)** — 출력을 봐도 상태/다른 키를 못 되돌림. ← CSPRNG 담당.
3. **전방 안전성 (forward secrecy)** — 현재 상태가 노출돼도 과거 키는 안전. ← 상태 갱신·과거 소거.

반례: 완벽한 CSPRNG라도 seed를 `12345`로 고정하면 역산은 불가능하지만 누구나 재현 → 뚫림. 무너진 건 ②가 아니라 ①. 거꾸로 엔트로피 충분해도 Mersenne Twister면 ②가 없어 뚫림.

→ 실무 키 생성이 **물리 엔트로피로 시드 → CSPRNG로 확장**하는 하이브리드인 이유가 이 세 조건을 한 번에 만족시키기 위해서.

## 5. LLM 비결정성을 난수에 쓸 수 있나 — 아니오

직관("비결정적이다 → 무작위다 → 엔트로피에 쓸 수 있다")이 어긋나는 지점. 무작위 소스의 가치는 비결정성이 아니라 ①근본적 예측불가 ②높은 엔트로피율 ③무편향 ④저비용·감사가능성.

- **샘플링(temperature/top-p)의 무작위성은 밑단 PRNG**가 굴리는 주사위 → `seed` 고정하면 재현됨. PRNG를 거대한 신경망으로 감싼 것일 뿐, 비용만 폭증.
- **부동소수점/GPU 지터**는 배치-분산 reduction 순서 차이에서 오는 것으로 **제거하려는 대상**(재현성 필요). 없앨 수 있다 = 근본적 무작위가 아님. (참고: 2025 "Defeating Nondeterminism in LLM Inference".)
- 설령 지터에서 엔트로피를 긁어도 소스로서 최악: 극심한 편향("1~100 랜덤" → 42/47에 몰림), 낮은 비트레이트, 천문학적 비용, 감사 불가.
- 사실 원하는 게 하드웨어 타이밍 지터라면 그건 **OS 엔트로피 풀이 이미 더 싸고 검증된 방식으로** 하고 있다.
- 역설: LLM의 유용성(재현성·제어가능성)이 높아질수록 비결정성은 사라진다 — 좋은 엔트로피 소스와 방향이 정반대.

→ 한 줄: ==LLM은 무작위를 "생성"하는 게 아니라 밑단 PRNG 주사위를 굴릴 뿐. 진짜 무작위가 필요하면 그 PRNG를 시드하는 물리 엔트로피(TRNG/QRNG)로 직행.==

## 곁가지 (탐색 후보)

- OS 엔트로피: 엔트로피 추정(estimation)과 부팅 직후 `getrandom()`이 블로킹되는 이유.
- randomness extractor(폰 노이만 추출기, 해시 화이트닝)로 편향 소스를 균등화하는 원리.
- 흥미로운 반대 방향: LLM을 난수 *소비자*가 아니라 수열의 편향·패턴을 탐지하는 *평가/공격 도구*로 쓰기.
- One-Time Pad가 정보이론적으로 안전한데도 실무에서 안 쓰이는 이유(키 관리·길이).
