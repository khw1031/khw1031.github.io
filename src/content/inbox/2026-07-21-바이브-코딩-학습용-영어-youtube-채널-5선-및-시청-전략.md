## TL;DR
- 바이브 코딩 영어 학습은 **도구 숙지(Duncan) → 전반 이해(Zinho) → 사업 적용(Jack)** 순서가 가장 효율적이라는 저자 주장.

## 큰 그림

```
[바이브 코딩 학습 여정]
│
├─ 1단계: 도구 익히기
│   └─ Duncan Rogoff (Claude Code 전문)
│
├─ 2단계: 생태계 넓히기
│   ├─ Zinho Automates (바이브 코딩·AI 앱·에이전트 전반)
│   └─ Stephen G. Pope (AI 자동화 시스템 구축)
│
└─ 3단계: 사업으로 연결하기
    ├─ Jack Roberts (AI 자동화로 실제 사업 운영)
    └─ Liam Ottley (AI 에이전시 모델·사업화)

시청법: 1.25배속 + 자동번역 자막 / 급하면 AI 요약부터
```

## 핵심
- 저자는 바이브 코딩 한국어 콘텐츠 다음 단계로 **영어 채널**을 권하며, 30분 내외의 긴 영상이 오히려 **막히는 과정까지 노출**되어 학습에 유리하다고 **주장**한다. 편집으로 실패를 숨긴 짧은 영상보다, 실제 문제 해결 과정이 보이는 영상이 입문자에게 더 높은 정보 밀도를 제공한다는 것이다. 5개 채널은 단순 목록이 아니라 **도구 → 생태계 → 사업화**라는 단계적 학습 경로를 형성하며, 저자 본인이 실제로 구독 중인 채널이라고 밝히고 있다.

## 깊이
- **[Duncan Rogoff — 도구 단계]** Claude Code만 전문으로 다루는 채널로, 저자는 **"입문이면 여기부터"**라고 명시했다. 특정 도구에 집중함으로써 초보자가 환경 설정·프롬프트 패턴 등 기초를 한 번에 익힐 수 있다는 **주장**이다. (검증 필요: 실제로 Claude Code 입문자에게 최적화된 콘텐츠인지는 시청자 평판을 확인해야 함)
- **[Zinho → Jack 연결 — 생태계→사업]** Zinho Automates는 바이브 코딩·AI 앱·에이전트를 폭넓게 다루므로 도구 숙지 후 시야를 넓히는 데 적합하고, Jack Roberts는 그 기술로 **실제 사업을 굴리는 과정**을 보여준다. 저자는 Duncan → Zinho → Jack 순서를 **"가장 덜 헤맨다"**고 추천했는데, 이는 기술→응용→비즈니스의 자연스러운 학습 곡선을 반영한 것으로 보인다.
- **[시청 전략]** 1.25배속 + 자동번역 자막은 영어 비원어민의 정보 흡수 속도를 높이는 실용적 방법이며, **"급하면 링크를 AI한테 던져 요약부터 받는다"**는 접근은 영상 전체를 보기 전에 핵심 구조를 먼저 파악하는 top-down 학습법에 해당한다. (저자 개인 경험 기반 **주장**, 일반적 효과는 불확실)

## 용어 풀이
- **바이브 코딩 (vibe coding)** — AI에게 자연어로 지시하며 코드를 생성하는 코딩 방식 / 비유: "네비게이션에 목적지만 입력하고 운전은 AI에게 맡기는 것" / 비유가 깨지는 지점: 실제 운전과 달리 AI가 잘못된 코드를 만들면 디버깅 책임은 사람에게 돌아온다.
- **AI 에이전시 모델 (AI agency model)** — AI 자동화 서비스를 대행·구축해주는 사업 형태 / 비유: "AI를 도구로 쓰는 게 아니라, AI 도입 자체를 상품으로 파는 것" / 비유가 깨지는 지점: 일반적인 대행업과 달리 AI 기술 변화가 빨라 지속 학습이 사업 유지의 전제조건이다.

## 핵심 시사점 / 판단
- **(저자 주장)** 편집되지 않은 긴 영상이 막히는 과정까지 보여주므로 학습에 더 유리하다.
- **(저자 주장)** Duncan → Zinho → Jack 순서가 입문자가 가장 덜 헤매는 경로다.
- **(검증 필요·불확실)** 각 채널의 현재 업데이트 빈도·콘텐츠 품질·구독자 규모 등 객관적 지표는 원문에 없음.
- **(검증 필요·불확실)** 1.25배속 + 자동번역 자막의 학습 효과가 일반적인지는 개인차 존재.
