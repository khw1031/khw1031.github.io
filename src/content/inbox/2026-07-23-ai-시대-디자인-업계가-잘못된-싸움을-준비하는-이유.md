## TL;DR
- AI가 구축을 무료에 가깝게 만들었지만, 제품은 '잘 못 만들어서'가 아니라 '잘못된 것을 만들어서' 죽는다 — 남은 유일한 스킬은 무엇을 만들지 결정하는 전략이다.

## 큰 그림
```
                ┌───────────────────────────┐
                │   시장의 심판 (현실)        │
                │ 무관심 · 대안 · 전환비용     │
                └────────────┬──────────────┘
                             │ 유일한 대응
                ┌────────────▼──────────────┐
                │       전략 (GUN)           │
                │ 누구를 위해, 왜 만들 것인가  │
                └────────────┬──────────────┘
                             │
            ┌────────────────┼────────────────┐
            ▼                ▼                ▼
     ┌───────────┐   ┌───────────┐   ┌───────────┐
     │ 크래프트   │   │ 표준       │   │ 테이스트   │
     │ (KNIFE)   │   │           │   │           │
     └───────────┘   └───────────┘   └───────────┘
     ↑ AI가 비용≈0   ↑ 중요하지만     ↑ 중요하지만
                       거리 못 닿음     거리 못 닿음
```

## 핵심
- (저자 주장) AI가 빌딩 비용을 거의 0으로 만들었지만, 업계 대화는 여전히 "어떻게 만들 것인가"에만 집중돼 있다. 크래프트·표준·테이스트는 모두 유효하나 근접 무기(knife)이고, 진짜 적인 시장은 원거리에서 무관심으로 제품을 죽인다.
- (저자 주장) 시장이 적인 이유는 협상하지 않고, 규칙을 알려주지 않으며, 피드백 없이 제품을 무시할 수 있기 때문이다. 유효 사거리로 대응할 유일한 무기는 '무엇을, 누구를 위해 만들 것인가'를 결정하는 전략(gun)뿐이다.
- 과거 빌딩 비용은 잘못된 아이디어를 자연스럽게 도태시켰으나, AI로 실험 비용이 사라지자 '빠르고 그럴듯한' 반복이 전략적 판단을 대체했다 — 비용이라는 강제 장치가 사라진 것이다.

## 깊이
- [전략 부재 사례] 저자가 컨설팅한 스타트업은 2.5년간 4번의 디자인 반복으로 보이스 UI·웹앱·AI 엔진을 갖췄지만, 누구를 위한 제품인지 설명하지 못했다. 엔지니어는 이를 "아픈 사람이 누군지도 모르는 신약 분자"에 비유했다. 공학은 뛰어나지만 시장 접점이 없었던 것. (2차, 저자 경험담)
- [비용-판단의 역설] Cloud Native Patterns에 따르면 실험 비용이 낮아지면 sunk-cost bias가 강해져 실패한 아이디어를 더 오래 끌고 간다. (2차 → 1차 패턴 카탈로그)
- [Jobs to Be Done] Christensen의 밀크셰이크 연구 — 소비자는 밀크셰이크를 '아침 통근의 지루함을 견디는 도구'로 고용했다. 경쟁자는 다른 밀크셰이크가 아니라 베이글과 지루함. AI는 이 '고용 맥락'을 알지 못한다. (저자 해석, 1차 연구 기반)

## 용어 풀이
- **Strategy-execution chasm** — '무엇을 만들지'와 '어떻게 만들지' 사이의 단절. 비유: 나침반 없이 지도만 상세히 그리는 것. 깨지는 지점: 실행 없이 전략만 있으면 공허한 슬라이드가 된다.
- **Sunk-cost pull** — 이미 투입한 비용 때문에 잘못된 방향을 유지하는 심리적 힘. 비유: 재미없는 영화를 티켓 값 때문에 끝까지 보는 것. 깨지는 지점: 개인 심리와 조직 역학은 다른 메커니즘으로 작동한다.
- **Taste as moat** — AI가 모방하기 어려운 미적·맥락적 판단력. 비유: 레시피는 복제해도 셰프의 간은 복제 못하는 것. 깨지는 지점: '테이스트' 정의가 저자마다 달라 검증이 어렵다.

## 시각 자료
| 구분 | 크래프트 (KNIFE) | 전략 (GUN) |
|------|-----------------|-----------|
| 질문 | 어떻게 만들 것인가 | 무엇을, 누구를 위해 만들 것인가 |
| 범위 | 근접 (내부 프로세스) | 원거리 (시장 현실) |
| AI 영향 | 비용 ≈ 0, 무한 생산 | 변화 없음 — 여전히 인간 판단 |
| 실패 시 | 품질 낮은 제품 | 아무도 원치 않는 제품 |

## 핵심 시사점 / 판단
- (저자 주장) AI 시대 제품팀의 핵심 역량은 빌딩이 아니라 '무엇을 만들지 결정하는 판단력'이다. 구축이 무료일수록 전략의 레버리지가 커진다.
- (저자 주장) 빌딩 비용은 과거 전략적 판단의 강제 장치였다. 그 장치가 사라진 지금, 의식적 전략 프로세스가 없으면 팀은 번아웃과 방향 상실로 향한다.
- (검증 필요·불확실) "시장이 유일한 적" 프레임은 B2B·내부 도구·규제 산업 등 시장 피드백이 느린 맥락에서도 동일하게 적용되는지 불확실.
- (원문에 없음) 저자는 '어떻게 전략을 수립할 것인가'에 대한 구체적 방법론을 제시하지 않는다.
