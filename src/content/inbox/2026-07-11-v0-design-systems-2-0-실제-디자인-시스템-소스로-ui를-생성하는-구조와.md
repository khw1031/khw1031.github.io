## 핵심

과거 v0 디자인 시스템(v1)은 shadcn 레지스트리를 매개로 토큰을 재정의하고 컴포넌트를 하나씩 등록하는 방식이었다(저자 주장). 이는 기존 스택을 shadcn으로 마이그레이션해야 하고, 시간이 지날수록 원본과 v0 사본이 달라지는 **drift** 문제가 있었다.

Design Systems 2.0(2025.06.25 발표)은 이 구조를 뒤집는다. 사용자의 시스템을 v0에 맞추는 대신, v0가 실제 시스템—패키지, 소스코드, 실제 앱, Figma, Storybook—을 입력받아 패턴을 학습한다. 핵심은 세 축이다: **Skill**(한 번 설정하면 팀 전체 채팅에서 재사용되는 설정 묶음), **빌드 시 소스 검증**(prop·token·icon이 실제 존재하는지 매 빌드 확인), **실 스타터**(패키지 설치·provider 마운트·폰트 배선이 완료된 상태에서 시작).

==이 세 축이 연결되면 v0는 '기억으로 흉내 내는 UI'가 아닌 '실제 컴포넌트로 조합한 UI'를 출력한다==. 저자는 5개 디자인 시스템을 동일 프롬프트에 적용한 데모로 이를 보여준다.

## 깊이

- **[검증 메커니즘]** v0는 매 빌드마다 실제 컴포넌트 소스를 read-only로 마운트하고 prop 존재 여부, token name 해석, icon export를 대조한다(저자 주장). 존재하지 않는 prop·icon을 참조하는 할루시네이션을 구조적으로 차단한다는 설명이다.

- **[데모 결과 비교]** 동일 프롬프트("GitHub 대시보드")에 5개 시스템을 각각 적용했다. Fluent는 검색 바의 비대칭 easing 애니메이션, Carbon은 다크모드에서 전체 토큰 팔레트 일괄 remap, Polaris는 테이블 헤더→bulk-actions 바 전환, Cloudscape는 모바일 breadcrumb 유지, Primer는 실제 GitHub과 거의 동일한 결과(저자 주장). 디자인 시스템 미적용 시 색상 스케일 불일치, 존재하지 않는 아이콘, 잔여 스피너가 발생했다.

- **[입력 소스 풍부성]** 저자는 "소스가 풍부할수록 출력이 좋아진다"고 주장한다. '실제 앱'이 provider·폰트·테마 배선의 실제 모습을 보여 가장 풍부한 소스라고 설명한다.

## 용어 풀이

- **Design System** — UI 일관성을 위한 컴포넌트·토큰·가이드라인 체계. 비유: 레고 세트+조립 설명서. 깨지는 점: 레고는 물리적 제약이 있으나 디자인 시스템은 코드·문서·Figma 등 다층 매체에 분산.
- **Skill (v0)** — 한 번 설정하면 이후 모든 채팅에 자동 적용되는 설정 묶음. 비유: 신입에게 건네는 컨벤션 cheat sheet. 깨지는 점: 사람은 맥락을 유연하게 해석하나 Skill은 규칙 기반으로 적용.
- **Registry (v1)** — 컴포넌트를 패키지 매니저처럼 등록·조회하는 저장소.
- **Drift** — 원본과 복제본이 시간 차에 따라 달라지는 현상.

## 시각 자료

| 디자인 시스템 | 특징적 디테일 (저자 관찰) |
|---|---|
| Fluent (MS) | 검색 바 비대칭 easing, 툴바 hover 시 Fluent Blue |
| Carbon (IBM) | 4단계 버튼 상태, 다크모드 전 토큰 remap |
| Polaris (Shopify) | 테이블 헤더→bulk-actions 전환, 체크 애니메이션 |
| Cloudscape (AWS) | collapsible 사이드바, 모바일 breadcrumb 유지 |
| Primer (GitHub) | 실제 GitHub과 거의 동일 |
| 미적용 | 색상 일관성 저하, 가짜 아이콘, 잔여 스피너 |

## 핵심 시사점 / 판단

- **(저자 주장)** 디자인 시스템 없이 생성하면 '기억의 천장'에 부딪히며, 실제 소스 연결 시 시스템 고유 디테일까지 재현 가능.
- **(저자 주장)** v1의 shadcn 의존·drift 문제를 v2에서 구조적으로 해결.
- **(검증 필요·원문에 없음)** 빌드 시 검증이 할루시네이션을 어느 수준까지 차단하는지 수치 데이터 없음. 대규모 시스템에서의 성능·정확도 미확인.
- **(불확실)** 5개 데모 결과가 모든 시스템·프롬프트 조합에서 동일 품질을 보장하는지 확인 안 됨.
