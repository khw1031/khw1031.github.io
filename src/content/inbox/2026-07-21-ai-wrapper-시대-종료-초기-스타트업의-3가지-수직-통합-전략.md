## TL;DR
- 단순 LLM wrapper는 끝났다. 살아남은 AI 스타트업은 워크플로우·데이터·도메인 지식을 통째로 소유하는 "수직 통합"으로 전환하고 있다.

## 큰 그림

```
[AI 스타트업 전략 변천]
 2023-24 Wrapper 시대 ──▶ 2025+ Verticalization 시대
 (AI as feature)         (AI as service)
      │                        │
  Jasper 붕괴          ┌───────┼───────────────┐
  (단일 태스크)        │       │               │
                  Approach1  Approach2     Approach3
                  원시지능   자동화 재설계  System of
                  대비       (Tomo)        Record→Action
                  (Blitzy)                 (Seso)
                       └───────┼───────────────┘
                         AGI 전환기 방어막
                         (워크플로우 소유 = 해자)
```

## 핵심
- NFX에 따르면 ChatGPT 등장 직후 "AI for X" 방식의 wrapper 스타트업(Jasper 등)이 대거 등장했으나, frontier lab이 자체 제품화(Claude Code, Claude Design 등)를 가속화하면서 단일 태스크 도구들은 생존 위협을 받았다. **저자는 주장한다**: 지능 자체가 상품이 되면서 "지능을 파는" 전략은 더 이상 방어막이 되지 못한다.
- 반대로 살아남은 회사들은 AI를 '기능'이 아닌 '서비스 전달 수단'으로 설계했고, 워크플로우·도메인 지식·데이터 축적 전체를 소유했다. 저자는 이를 "공장이 곧 제품(The factory is the product)"이라 표현한다.
- 초기 스타트업이 택하는 수직 통합은 3가지 형태를 띠며, 공통점은 **단일 모델 의존을 넘어선 오케스트레이션과 도메인 잠금**이다.

## 깊이
- **[Approach 1 — 원시 지능에 대항: Blitzy]** 기업 소프트웨어 개발에서, Blitzy는 기존 contractor 군단에 도구를 판매하는 대신 AI 자체를 contractor 대체 서비스로 만들었다(저자 주장). 코드베이스 전체의 knowledge graph + 태스크 분해 + 다중 모델 쿼리 오케스트레이션 레이어를 구축해 SWE-Bench Pro 66.5%로 1위를 기록했다고 원문은 밝힌다. 핵심 시사점: 복잡한 엔터프라이즈 맥락에서는 **맥락 소유 > 원시 지능**.
- **[Approach 2 — 자동화로 비즈니스 재설계: Tomo]** 모기지 업계에 AI를 "덧입히지" 않고, underwriting·영업·운영 전체를 자동화 중심으로 재설계했다. 그 결과 대출 담당자당 생산성이 경쟁사 대비 유의미하게 높고, 77%의 고객이 기존 사업자보다 낮은 금리를 받는다고 원문은 **주장**한다. 저자는 "기존 워크플로우에 소프트웨어를 끼워 넣는 SaaS 방식에서 벗어났다"고 평가한다.
- **[Approach 3 — System of Record → System of Action: Seso]** 미국 농업의 H-2A 비자·계절 인력 관리가 법事務所·Office·종이에 흩어진 혼란을 중앙 DB로 먼저 정리(System of Record)한 뒤, AI로 물류·인사 결정·비즈니스 인사이트를 실행하는 System of Action으로 진화했다. 저자는 "Record를 가지면 다음 단계는 반드시 Action"이라 일반화한다.

## 용어 풀이
- **Wrapper** — 기존 LLM API 위에 얇은 UI/프롬프트만 씌운 제품 / 비유: "라면 스프만 바꿔 파는 가게" / 깨지는 지점: 오케스트레이션·파인튜닝·자체 데이터 파이프라인이 추가되면 더 이상 wrapper가 아님.
- **Verticalization** — 특정 산업의 워크플로우·데이터·판매망을 통째로 소유하는 전략 / 비유: "커피 원두 농장부터 카페까지" / 깨지는 지점: 초기 스타트업은 자본 제약으로 전 단계 동시 소유가 불가능하므로, 실제로는 핵심 한 단계를 먼저 잠근 뒤 확장.
- **System of Record / System of Action** — 기록 저장소(디지털 파일 캐비닛) vs 그 데이터로 자동 의사결정·실행을 하는 체계 / 비유: "가계부 → 자동 투자 어드바이저".
- **Frontier lab** — 최상위 기초 모델을 직접 학습시키는 조직(OpenAI, Anthropic, Google 등).

## 시각 자료
| 접근 | 대표 사례 | 산업 | 방어막 원천 | 원문 근거 지표 |
|---|---|---|---|---|
| 1. 원시 지능 대비 | Blitzy | 엔터프라이즈 코딩 | 코드베이스 knowledge graph + 오케스트레이션 | SWE-Bench Pro 66.5% (1위) |
| 2. 자동화 재설계 | Tomo | 모기지 | 영업~언더라이팅 전 과정 자동화 | 77% 고객 더 낮은 금리 |
| 3. Record→Action | Seso | 농업 HR (H-2A) | 파편화된 비자·인력 프로세스 중앙화 | 원문에 수치 없음 |

## 핵심 시사점 / 판단
- (저자 주장) Wrapper 세대는 끝났고, 수직 통합이 AGI 전환기에도 유효한 소수의 진짜 방어막이다.
- (저자 주장) Frontier lab의 자체 제품화(Claude Code 등)는 2년 전 wrapper였던 범주를 흡수하고 있어, 단일 태스크 제품은 구조적으로 취약하다.
- (저자 주장) 대기업 + frontier lab 제휴(Freshfields × Anthropic 등)는 비용·토큰 비용·보유 지식 공유 저항 때문에 AI-native 스타트업에 비해 민첩성이 낮을 수 있다.
- (검증 필요·불확실) Blitzy의 $1.4B 밸류에이션과 SWE-Bench Pro 1위 수치는 원문 주장이며 독립 검증이 필요.
- (검증 필요·불확실) Anthropic 매출 run rate $14B→$47B 수치도 2차 보도(Newcomer·VentureBeat) 인용으로, 공식 공시 여부는 불확실.
- (검증 필요·불확실) "일반 에이전트는 wrapper 2.0"이라는 Reddit 기반 언급은 의견이며 실증 근거 없음.
