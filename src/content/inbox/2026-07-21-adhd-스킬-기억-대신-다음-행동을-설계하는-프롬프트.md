## TL;DR
- 이 스킬의 목표는 사용자가 내용을 "기억하게" 만드는 게 아니라, 기억하지 않아도 **다음 행동으로 넘어가게** 만드는 것이다.

## 큰 그림
```
[일반 AI 답변]                    [ADHD 스킬 답변]
   정보 나열                          행동 설계
   ┌─────────┐                       ┌─────────┐
   │ 요약·설명 │ ── 사용자가          │ 다음    │ ── 사용자가
   │ 기억 burden │    기억해야        │ 한 걸음 │    기억 없이
   └─────────┘    실행 가능           └─────────┘    바로 실행
       ↓                                ↓
   "이해했다"                      "이제 이걸 한다"
   (실행 누락 위험 ↑)              (실행 마찰 ↓)
```

## 핵심
- 저자(@aicoffeechat)는 이 스킬을 "기억하게 만드는 답변"이 아니라 "기억하지 않아도 다음 행동을 할 수 있게 만드는 답변"으로 규정한다. 즉 **인지 부하를 사용자에게 떠넘기지 않는 설계**가 핵심이다.
- 저자는 핵심을 **"짧게"가 아니라 "움직이게"**라고 요약한다. 이는 요약을 극단적으로 줄이는 것(minimalism)과 실행 가능한 다음 동작을 제시하는 것(actionability)이 서로 다른 축임을 주장하는 대목이다.
- GitHub(`ayghri/i-have-adhd`)에 공개된 스킬이며, 현재 "엄청 바이럴되고 있다"고 저자는 주장한다. 다만 바이럴의 구체적 수치(star 수, 포크 수 등)는 원문에 제시되지 않아 **불확실**.

## 깊이
- **[짧음 vs 움직임]** "짧게"는 토큰/문장 수를 줄이는 형식적 제약이고, "움직이게"는 답변 끝에 사용자가 취할 구체적 다음 동작을 부착하는 구조적 제약이다. 저자에 따르면 후자가 이 스킬의 지향점이다. (원문에 구체적인 프롬프트 지침은 포함되지 않음 — SKILL.md 원본 확인 필요.)
- **[ADHD 컨텍스트]** ADHD는 작업 기억(working memory)과 실행 기능(executive function)의 어려움과 연관된다는 점이 널리 알려져 있다. 저자는 이 점을 전제로, "기억"을 요구하지 않는 답변 형식이 ADHD 사용자에게 더 적합하다고 **주장**한다. 임상적 근거는 원문에 없음.
- **[스킬(Skill)이란]** 프롬프트 엔지니어링 맥락에서 "스킬"은 특정 역할·톤·제약을 담은 재사용 가능한 시스템 지시문을 뜻하는 경우가 많다. 이 저장소 역시 `skills/i-have-adhd/SKILL.md` 경로로 보아 단일 프롬프트 모듈로 추정되나, 실제 구조는 **원문에 없어 확인 필요**.

## 용어 풀이
- **스킬(Skill)** — AI에게 특정 행동 양식을 지시하는 재사용 가능한 프롬프트 파일. / 비유: 배우에게 건네는 "배역 메모". / 깨지는 지점: 실제 코드·도구·API 호출을 포함하는 에이전트 스킬과 혼동되기 쉬움. 이 저장소가 어느 쪽인지는 원문만으로 불확실.
- **ADHD** — 주의력결핍 과잉행동장애. 이 스킬은 임상적 개입이 아니라 **사용자 경험(UX) 차원의 프롬프트 설계**로 접근한다. 의학적 조언으로 오독하면 안 된다.
- **바이럴** — 소셜 미디어에서 빠르게 확산되는 현상. 저자의 주관적 표현이며 수치 근거는 원문에 없음.

## 핵심 시사점 / 판단
- **(저자 주장)** 이 스킬의 진짜 차별점은 "짧음"이 아니라 "다음 행동 제시"에 있다. ADHD가 아닌 사용자에게도 적용 가능한 보편적 UX 원칙일 수 있다.
- **(저자 주장)** "기억하게 만드는 답변"은 사용자에게 인지 부하를 전가하는 설계다.
- **(검증 필요·불확실)** 실제로 SKILL.md가 어떤 구체적 규칙(예: 항상 한 줄의 next-action을 붙인다, 체크박스 형태로 준다 등)을 담고 있는지는 원문에 없어 GitHub 원본을 직접 확인해야 한다.
- **(검증 필요·불확실)** "바이럴"의 근거(star, 포크, Threads 반응 규모)는 원문에 제시되지 않았다.
- **(경계)** 이 스킬이 ADHD에 대한 임상적 근거에 기반하는지, 아니면 경험적 휴리스틱인지는 원문만으로 판단 불가. 의학적 개입으로 혼동하지 말 것.

- **Q2(왜·어떻게)** SKILL.md는 구체적으로 어떤 프롬프트 규칙으로 "다음 행동"을 강제하는가? (예: 접미사 템플릿, JSON action 스키마, 체크박스 포맷 등)
- **Q3(경계)** 이 스킬을 과도하게 적용하면 답변이 지나치게 지시적(paternalistic)이 되거나, 복잡한 판단이 필요한 주제에서 "다음 한 걸음"으로 환원되는 위험은 없는가?
