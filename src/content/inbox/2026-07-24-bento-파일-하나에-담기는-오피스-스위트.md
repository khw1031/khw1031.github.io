## TL;DR
- 문서 데이터와 편집 앱 자체를 하나의 `.bento.html` 파일에 담아, 브라우저만 있으면 설치·계정 없이 열고 편집하고 발표할 수 있고, 평문 JSON 덕분에 AI 에이전트가 파일을 직접 수정하는 PowerPoint 대체재다.

## 큰 그림
```
               Bento / Suite (MIT, 로컬 우선)
                          │
   ┌──────────────────────┼──────────────────────┐
   │                      │                      │
 단일 파일          AI 친근 포맷            로컬 우선 협업
 .bento.html       <script> 평문 JSON         AES-GCM E2E

        │                      │                      │

        │                      │                      │

   로드맵: Slides(현재) → Docs → Sheets (각각 독립 .bento.html)
```

## 핵심
파일 하나가 문서이자 앱이 되는 게 Bento의 중심축이다. `.bento.html` 안에는 편집기 코드뿐 아니라 슬라이드 데이터가 평문 JSON으로 들어 있어 브라우저가 파일을 열면 곧바로 편집기가 실행되고, 저장하면 파일 자체에 데이터가 다시 기록된다. 이 구조가 계정·클라우드·설치 절차를 없애는 동시에, JSON이 노출돼 있다는 점이 AI 에이전트의 직접 편집 경로로 이어진다. 즉 "문서가 곧 앱"이면서 "문서가 곧 기계가 읽는 데이터"인 것이다.

이 AI 친화성은 기존 `.pptx`를 단순 변환하지 않고 AI가 Bento 기능(Morph·실시간 차트)으로 다시 작성하는 활용으로 확장된다. 그리고 단일 파일이면서도 파일 사본 공유만으로 협업 초대가 되도록, 협업 키를 파일 자체에 넣고 자체 CRDT로 온·오프라인 편집을 병합한다. 로컬 우선 원칙이 AI 친화 포맷과 만나 "파일 주고 Claude에게 고쳐달라 하면 끝"이라는 워크플로우를 만든다.

## 깊이
- [단일 파일-압축/서명] ~570KB 파일에는 앱이 base64 blob로 들어가고 브라우저 `DecompressionStream`으로 압축 해제해 외부 파일 의존 없이 실행한다(저자(제작자) HN 댓글, 2차). 저장는 File System Access API로 같은 파일에 재기입하고 미지원 시 일반 다운로드로 대체하며, 모든 업데이트는 ECDSA로 서명한다(저자 주장). reveal.js·echarts·GSAP에서 시작했으나 크기·동작 문제로 애니메이션·차트를 직접 재구현했다(저자 주장).
- [AI 포맷-json] 데이터는 파일 상단 `<script type="application/bento+json" id="bento-doc">` 평문 JSON. `window.bento.doc`로 읽고 `window.bento.loadDoc(json)`으로 쓴다. 앱 코드를 건드리지 않고 JSON만 고쳐 슬라이드·차트·발표자 노트 추가 가능. 파일 첨부 안 되는 챗봇은 JSON 복붙, Ollama/llama.cpp/LM Studio면 편집도 로컬 가능. `agents.md`로 에이전트 사용 안내(본문, 2차).
- [협업-CRDT/릴레이] 라이브 세션 시 같은 파일 사본을 연 사용자가 참여. AES-GCM 종단간 암호화, 중계 서버는 암호문만 처리(블라인드 릴레이, Cloudflare Durable Objects). 자체 CRDT가 실시간·오프라인 변경 양방향 병합. 사용자 키로 읽기전용 권한·권한 철회(저자 주장). 트래픽 부하·운영비 신뢰성은 미확실.

## 용어 풀이
- CRDT(Conflict-free Replicated Data Type) — 여러 곳에서 동시에 편집해도 나중에 합칠 때 충돌 없이 섞이는 자료구조 / 구글 문서의 동시편집 같은 것 / 비유가 깨지는 지점: 네트워크 통신·캐시 비용과 "수학적 보장"이 실사용 UX의 매끄러움과 항상 일치하진 않는다.
- Morph 전환 — 같은 id 요소가 슬라이드 간 자연스럽게 이동·변형 / 영화 컷전환 대신 같은 인물이 카메라 이동 / 비유 깨짐: 별도 타임라인 없이 구성된다고 하지만 복잡한 키프레임 제어엔 한계.
- DecompressionStream — 브라우저 내장 압축해제 API / zip 풀기 기능이 브라우저에 내장된 것 / 깨짐: 구형 브라우저 지원 범위.
