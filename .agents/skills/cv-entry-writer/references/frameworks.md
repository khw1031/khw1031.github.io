# Résumé entry frameworks

The site model (narrative + separated impact + optional Why) is a blend of the
standard frameworks below. Use these to decide how to phrase a given entry.

## Menu

| Framework | Shape | Best for |
| --- | --- | --- |
| STAR | Situation · Task · Action · Result | Interview stories; too long for a résumé line |
| CAR / PAR | (Context ·) Problem → Action → Result | Résumé default; short, clear cause→effect |
| XYZ (Google) | "Accomplished Y, by doing Z, measured by X" | Impact/number-first; strong for lead/senior |
| What/How/Impact | Definition → method → result | Simple, but the labels read as artificial |

## This site's blend

- **Narrative** absorbs Problem/Why (Situation) + Action (what/how) into flowing prose.
- **Impact** is pulled out as XYZ-style quantified results, kept separate and scannable.
- **Why** is the CAR "Problem/Context" — a single opening sentence, optional, folded
  into the narrative (never a separate label). Most valuable for lead/ownership scope.

## Voice: 개조식 (명사형 종결)

Korean résumé standard.

- Noun/nominalized endings: 개발 · 구축 · 운영 · 설계 · 확보 · 단축 · 절감 · 전환.
- No subjects, no sentence endings (`-했다`, `-합니다`, `-함`, `-음`).
- Connectives compressed with `·`, `/`, `→` where it aids scanning.
- Numbers first in impact: "예상 2~3개월 → 21일 (약 62%↓)".

### Convert examples

- ✗ "…웹뷰를 Claude Code 기반으로 개발했다." → ✓ "Claude Code 기반 AI 워크플로우로 웹뷰 개발 (3인 팀)"
- ✗ "…자산으로 전환했다." → ✓ "…동일 절차로 사용 가능한 자산으로 전환"
- ✗ "62% 일정 단축을 달성했다." → ✓ "예상 2~3개월을 21일로 단축, 약 62% 일정 절감"
